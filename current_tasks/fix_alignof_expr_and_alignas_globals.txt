Fix __alignof__(expr) returning hardcoded 8 and _Alignas on global variables

Problem 1: __alignof__(variable) always returns 8 because the parser
discards the expression operand and returns IntLiteral(8). GCC's
__alignof__ on a variable should return the alignment of the variable's
type (or explicit _Alignas alignment if specified).

Problem 2: _Alignas on global variable declarations may not correctly
propagate alignment to the ELF section, causing misaligned globals.

Fix:
- Add AlignofExpr(Box<Expr>, Span) variant to AST
- Parse __alignof__(expr) correctly when it's not a type
- Lower AlignofExpr by looking up the expression's type alignment
- Fix const-eval for AlignofExpr in both sema and IR paths

Tests affected: gcc_torture_stkalign, compiler_suite_0057_0008
