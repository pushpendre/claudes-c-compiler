Fix ARM MOV large immediate encoding for libpng build

Problem: The ARM backend emits bare `mov reg, #imm` instructions for large
immediates that cannot be encoded in a single AArch64 MOV instruction.
AArch64 MOV (alias for MOVZ/MOVN) can only encode a 16-bit value optionally
shifted by 0/16/32/48 bits.

Affected locations in codegen.rs:
1. emit_switch_case_branch (line ~1315): mov x1, #{case_val}
2. emit_add_imm_to_acc (line ~1891): mov x1, #{imm}
3. emit_set_return_f64_second (line ~3198): mov x0, #{bits}
4. emit_set_return_f32_second (line ~3226): mov w0, #{bits}
5. emit_load_imm64 (line ~601): first mov should be movz
6. load_large_imm (line ~583): doesn't handle negatives or >32-bit values

Fix: Use emit_load_imm64 / movz+movk sequences for all large immediates,
and fix emit_load_imm64 to use movz for the first instruction.
