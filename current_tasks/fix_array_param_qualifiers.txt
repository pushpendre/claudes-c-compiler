Task: Fix C99 array parameter qualifiers (restrict, static, const, volatile)

Bug: The parser didn't handle type qualifiers (restrict, const, volatile) and
'static' inside array brackets in function parameter declarations. This caused
parse errors for:
    void foo(int a[restrict n])
    void bar(int b[static restrict 10])

Fix: Added skip_array_qualifiers() helper that skips static/const/volatile/
restrict/_Atomic inside array brackets before parsing the size expression.
Applied to all array bracket parsing locations in declarators.rs.

Status: In progress
