Fix block-scope extern declarations with __attribute__((error(...))) not being tracked in lowering

Problem:
The Linux kernel's BUILD_BUG_ON macro expands to:
  do {
    extern void __compiletime_assert_NNN(void) __attribute__((error("...")));
    if (!(condition)) __compiletime_assert_NNN();
  } while(0)

This creates a block-scope extern function declaration with the error attribute.
The lowering phase collects error_functions only from file-scope ExternalDecl::Declaration
items (lowering.rs:622), but block-scope function declarations go through
try_lower_block_func_decl (stmt_init.rs:63), which does NOT check for the error attribute.

Result: Calls to __compiletime_assert_NNN are emitted as real function calls
instead of being replaced with Unreachable, causing hundreds of linker errors.

Fix: In try_lower_block_func_decl, propagate error_attr and noreturn attributes
from block-scope function declarations to error_functions and noreturn_functions sets.

This is a lowering-phase fix (IR generation), NOT a sema fix for -Wreturn-type.
