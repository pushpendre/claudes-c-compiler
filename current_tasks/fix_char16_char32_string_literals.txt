Task: Implement proper u"" (char16_t) and U"" (char32_t) string literal support
Status: in_progress

Problem:
- u"..." string literals are currently routed to the narrow (char) string path in the lexer
- This means u"abcd" produces an array of 8-bit chars instead of 16-bit char16_t values
- The test compiler_suite_0057_0082 fails because of this

Fix Plan:
1. Add Char16StringLiteral token variant to lexer
2. Add Char16StringLiteral AST variant to parser
3. Route u"..." through the new path in the lexer
4. Add IR Char16String global init variant (Vec<u16>)
5. Update IR lowering for char16_t local and global array init
6. Update backend emission to emit .short (2-byte) values for char16_t strings
7. Handle type checking to return unsigned short* for char16_t strings
