Task: Fix parenthesized declarator parsing
Status: IN PROGRESS

Problem:
The parser only handles one level of parenthesization around declarator names, e.g., (name).
It does not handle:
- Nested parenthesization: int ((x)) = 1, int (((x))) = 1
- Parenthesized function declarations: int(fn(void))
- Parenthesized declarators with function suffixes: int((return_3))(void)
- Complex declarators: long(*two_pointers(double val, double(*d)))

This causes parser errors on valid C code like:
  parser error: expected Semicolon, got LParen

Tests affected:
- wacc_chapter_14_valid_declarators_declarators
- wacc_chapter_15_valid_libraries_return_pointer_to_array
- compiler_suite_0048_0009 and others

Fix:
Rewrite parse_declarator to recursively handle parenthesized declarators
as per the C standard grammar for direct-declarator:
  direct-declarator:
    identifier
    ( declarator )
    direct-declarator [ constant-expression_opt ]
    direct-declarator ( parameter-type-list )
